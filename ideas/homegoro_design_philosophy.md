# 誉め殺し格ゲー — 思想設計書

## 0. コンセプト一行

**「相手を褒めて倒す格闘ゲーム」**

褒め言葉が技になる。褒め方の巧さがダメージになる。
最も美しく相手を称えた者が、最も残酷に相手を破壊する。

---

## 1. Why — なぜこのアプリが存在するのか

### 1-1. 根底にある矛盾の構造

このアプリの核は **「褒める＝攻撃」** という認知の逆転にある。

人間社会では「褒める」は善行であり、「攻撃」は悪行。
この2つが同一の行為として結合することで、
脳がバグる。その**認知的不協和そのものが面白さ**になる。

```
通常の格ゲー:  罵倒 → ダメージ（直感的）
誉め殺し格ゲー: 褒め → ダメージ（逆転的）
```

これは単なるネタではない。
「誉め殺し」は日本語として実在する概念であり、
**褒めることの中に潜む暴力性** を誰もが薄々感じている。
このアプリはそれを可視化・ゲーム化する。

### 1-2. SlapMacとの構造比較

| | SlapMac | 誉め殺し格ゲー |
|--|---------|-------------|
| 逆転構造 | 叩く（暴力）→ 喘ぎ声（快楽） | 褒める（善意）→ ダメージ（攻撃） |
| 禁忌 | 高価なMacBookを叩く | 人を「褒め殺す」 |
| センサー | 加速度（物理） | マイク＋音声認識（言語） |
| 深さ | 叩く強さの1軸 | 語彙力・創造性・タイミングの多軸 |
| リプレイ性 | 叩く→音が出る（飽きやすい） | 毎回違う褒め言葉（飽きにくい） |
| 文化性 | 万国共通（物理） | 言語依存だが深い（言葉遊び） |

**SlapMacを超えるポイント:**
- SlapMacは「物理の1軸」だが、誉め殺しは「言語の無限軸」
- 毎回異なる褒め言葉＝毎回異なるコンテンツが自動生成される
- 「上手い褒め」と「下手な褒め」で差が出る＝スキルゲームになる

### 1-3. 世の中に対するメッセージ

表向きは完全なネタアプリ。しかし体験した後に残るのは：

- **「褒める」行為の奥深さへの気づき** — 褒めるって意外と難しい
- **言葉の力の再認識** — 同じ「すごい」でも文脈で威力が変わる
- **コミュニケーションの遊び化** — 褒め合うことが娯楽になる

IT業界の「バカバカしいアイデアが勝つ」という法則に乗りつつ、
**実は人間のコミュニケーション能力を鍛えるアプリ** でもある。
この二重構造が、ネタで終わらない長期的な価値を生む。

## 2. 体験設計 — 褒める行為の感情曲線

### 2-1. ユーザーが辿る感情の旅

```
① 困惑（え、褒めて攻撃？）
    ↓
② 試行（とりあえず「すごいね」と言ってみる → 弱い技が出る）
    ↓
③ 発見（「君の瞳は砂漠のオアシスだ」→ 大技が炸裂！）
    ↓
④ 没入（もっと上手く褒めたい → 語彙力フル回転）
    ↓
⑤ 爆笑（自分の褒め言葉のバカバカしさに笑う）
    ↓
⑥ 共有（この褒め言葉見てくれ → SNSシェア）
    ↓
⑦ 再戦（次はもっと上手く褒める → リプレイ）
```

### 2-2. コアループ

```
褒め言葉を考える → 声に出す/入力する → AI判定 → 技が発動 → ダメージ演出
     ↑                                                    |
     └──────────── 「もっと上手く褒めたい」 ←──────────────┘
```

**このループが回り続ける理由:**
- 褒め言葉は無限にある＝入力の天井がない
- 「上手い褒め」の基準が曖昧＝毎回新しい発見がある
- 対人戦なら相手の反応というフィードバックもある

### 2-3. 3つのプレイモード構想

| モード | 概要 | 感情設計 |
|-------|------|---------|
| **一人修行** | AI相手に褒め技を磨く | 「俺こんな褒め方できるんだ」という自己発見 |
| **ローカル対戦** | 同じ画面で交互に褒め合う | 友達の予想外の褒め言葉に爆笑 |
| **オンライン対戦** | 知らない人とリアルタイム褒め合い | 見知らぬ人の語彙力に感動/衝撃 |

## 3. バイラル構造 — 褒め言葉がコンテンツになるメカニズム

### 3-1. なぜ褒め言葉はバズるのか

SlapMacのバイラルは「叩いてる動画」。映像がコンテンツ。
誉め殺し格ゲーのバイラルは **「褒め言葉そのもの」** がコンテンツ。

```
SlapMac: 映像コンテンツ（撮影が必要）
誉め殺し: テキストコンテンツ（スクショで拡散可能）＋映像もいける
```

**テキストで拡散できるアドバンテージ:**
- 動画を撮る手間がない
- スクリーンショット1枚でバズれる
- Xとの相性が最高（テキスト主体SNS）
- 翻訳すれば他言語でもバズる

### 3-2. バイラルが発生する5つの瞬間

1. **名言爆誕の瞬間** — 「今世紀最高の褒め言葉が出た」→ スクショ共有
2. **予想外の大技発動** — 適当な褒めが超必殺技に化ける → 動画映え
3. **対戦のドラマ** — 逆転KOの瞬間 → リプレイ動画
4. **AI判定への突っ込み** — 「なんでこれがS判定なんだよ」→ 議論が生まれる
5. **有名人の参戦** — 配信者/YouTuberがプレイ → 視聴者が褒め言葉を投稿

### 3-3. シェアされるコンテンツの種類

| コンテンツ | 形式 | 拡散先 |
|----------|------|-------|
| 「今日の最強褒め」スクショ | 画像 | X / Instagram |
| 対戦のハイライト動画 | 短尺動画 | TikTok / Reels |
| 褒め言葉ランキング | リスト | X / Reddit |
| 友達とのローカル対戦動画 | 動画 | TikTok / YouTube Shorts |
| AI判定への不満/爆笑 | テキスト | X |

### 3-4. UGCが自動生成される構造

このアプリの最大の武器は **ユーザーがプレイするだけでコンテンツ（褒め言葉）が生まれる** こと。

SlapMacは「叩く」で出力は固定の音声。100人が使っても同じ音。
誉め殺し格ゲーは100人が使えば100通りの褒め言葉が生まれる。
**プレイヤー全員がコンテンツクリエイター**になる。

## 4. ゲームデザイン — 褒めの駆け引きと戦略性

### 4-1. 褒め言葉の評価軸

ただの音声認識ではない。褒め言葉を**多軸で評価**し、技の威力に変換する。

| 評価軸 | 説明 | 例 |
|-------|------|---|
| **語彙力** | 珍しい単語・表現を使っているか | 「すごい」(E) vs「比類なき」(S) |
| **具体性** | 抽象的でなく具体的に褒めているか | 「いいね」(E) vs「その眉毛の角度が完璧」(A) |
| **創造性** | 比喩・詩的表現・予想外の切り口 | 「君は太陽だ」(B) vs「君の存在は月曜の朝に届くドーナツだ」(S) |
| **感情強度** | 声のトーン・抑揚（音声入力時） | 棒読み(E) vs 魂の叫び(S) |
| **文脈力** | 対戦相手やシチュエーションに合った褒め | テンプレ(E) vs その場の即興(S) |

### 4-2. ランク → 技の対応

```
E ランク（とりあえず褒めた）  → 小パンチ「おべっか」
D ランク（まあまあの褒め）    → 中パンチ「お世辞アッパー」  
C ランク（なかなかの褒め）    → 大パンチ「賞賛ストレート」
B ランク（良い褒め）         → 必殺技「誉め殺しブロー」
A ランク（素晴らしい褒め）    → 超必殺技「言霊カタストロフィ」
S ランク（神の褒め）         → 一撃必殺「涙腺崩壊フィニッシュ」
```

### 4-3. 戦略性を生む仕組み

単に「上手く褒める」だけでは浅い。駆け引きを生む仕組みが必要。

**① 褒めジャンルのじゃんけん構造**

| ジャンル | 強い対象 | 弱い対象 |
|---------|---------|---------|
| 外見を褒める | 内面褒めに強い | 才能褒めに弱い |
| 内面を褒める | 才能褒めに強い | 外見褒めに弱い |
| 才能を褒める | 外見褒めに強い | 内面褒めに弱い |

**② コンボシステム — 褒めの連鎖**

同じ対象を異なる角度から連続で褒めるとコンボ成立。

```
「君の笑顔は素敵だ」（外見） → 単発ヒット
「君の笑顔は素敵だ」→「その笑顔で何人救われたか」（外見→内面） → 2コンボ！
→「それが才能だよ」（内面→才能） → 3コンボ！！
```

**③ カウンター — 褒め返し**

相手の褒めに対して「いやあなたこそ」と褒め返すとカウンター発動。
相手の褒めの威力を吸収して倍返しする防御技。

**④ 時間制限のプレッシャー**

各ターン制限時間あり。考える時間が短いほどボーナス。
「瞬時に上手い褒めが出る」= 真のコミュ力の証明。

### 4-4. キャラクター（褒めスタイル）構想

プレイヤーが選ぶ「褒めスタイル」がキャラクター性を生む。

| スタイル | 特徴 | 得意技 |
|---------|------|-------|
| **詩人タイプ** | 比喩と美辞麗句で攻める | 創造性ボーナス大 |
| **おかんタイプ** | 日常的で温かい褒め | 具体性ボーナス大 |
| **ホストタイプ** | 大げさで情熱的 | 感情強度ボーナス大 |
| **評論家タイプ** | 分析的で知的な褒め | 語彙力ボーナス大 |
| **子供タイプ** | 素朴で直球な褒め | 予想外ボーナス（ランダムでS化） |

## 5. 音・演出・トーンの哲学

### 5-1. トーン設計 — 「大真面目にバカバカしい」

このアプリのトーンは **「格ゲーとして完全に本気」** であるべき。

- UI/UXは正統派格ゲーのクオリティ
- HPバー、KO演出、コンボカウンターは全て本物の格ゲー品質
- 「ネタアプリ」感を出さない＝ギャップが笑いを生む

```
× ポップでゆるい → 「ネタアプリだな」で終わる
○ ハードコア格ゲー風 → 「え、マジで本格的…で褒めるの？」が面白い
```

### 5-2. サウンドデザイン

| 場面 | 音の方向性 |
|------|----------|
| タイトル画面 | 重厚なオーケストラ（ストリートファイター風） |
| ターン開始 | 「PRAISE YOUR OPPONENT」（渋いナレーション） |
| Eランク褒め | しょぼい効果音（ぺち） |
| Sランク褒め | 天から光が差す荘厳なSE + 画面全体が金色に |
| コンボ発動 | 格ゲーのヒットストップ + コンボカウント音声 |
| KO | 「PRAISED TO DEATH」（大文字で画面に） |
| 勝利演出 | 負けた側が感動の涙を流す演出 |

### 5-3. 視覚演出

**Eランク（弱パンチ）:**
- 小さなハートが1つ飛ぶ
- 相手キャラがちょっと照れる

**Sランク（一撃必殺）:**
- 画面が金色に染まる
- 桜吹雪 or 光の柱が降り注ぐ
- 相手キャラが膝から崩れ落ちて号泣
- 「EMOTIONAL DAMAGE」の文字が格ゲー風に炸裂
- 画面にヒビが入る（SlapMacオマージュ）

### 5-4. 名前の候補

| 名前 | 由来 | 良い点 | 懸念 |
|------|------|-------|------|
| **HoméGoroshi** | 誉め殺し（ローマ字） | 日本語のインパクト、海外で「何これ？」感 | 発音しづらい |
| **PraiseKO** | Praise + KO | グローバルで通じる、格ゲー感 | やや普通 |
| **KillWithKindness** | 英語の慣用句「親切で殺す」 | 意味が完璧、英語圏でバズる | 長い |
| **HOMEGORO** | 誉め殺し → ホメゴロ | キャッチー、キャラ名っぽい | |
| **PraiseOrDie** | 褒めるか死ぬか | 緊迫感、格ゲー感 | やや物騒 |

## 6. 技術的実現性の概観（実装設計ではない）

### 6-1. 褒め判定に必要なもの

核心の問いは **「褒め言葉の質をどう数値化するか」**。

**選択肢A: AIオンライン判定（LLM）**
- ChatGPT/Claude APIで褒め言葉を評価
- 最も柔軟で精度が高い
- ネットワーク遅延・コストの問題

**選択肢B: ローカルルールベース**
- 語彙辞書 + 正規表現 + スコアリング
- 即座に結果が出る
- 精度に限界がある

**選択肢C: ハイブリッド**
- 基本はローカルで即座にランク判定
- バトル後にAIが「名勝負解説」として詳細評価
- 遅延を感じさせずに深い判定ができる

### 6-2. 入力方式

| 方式 | メリット | デメリット |
|------|---------|----------|
| 音声入力 | 声のトーンも評価できる、動画映え | 静かな場所で使えない |
| テキスト入力 | どこでも使える、シェアしやすい | 声の熱量が伝わらない |
| 両対応 | 柔軟 | 評価基準の統一が難しい |

### 6-3. プラットフォーム

- Phase 1: Webアプリ（スマホブラウザ対応）
- Phase 2: バズったらiOS/Androidネイティブ
- Phase 3: 対戦機能はWebSocketでリアルタイム

## 7. 残された問い — 実装前に決めるべきこと

1. **1人プレイ vs 対戦、どちらを先に作るか？**
   - 1人プレイの方が開発は早い
   - 対戦の方がバイラルする
   
2. **AI判定のリアルタイム性はどこまで必要か？**
   - 叫んだ瞬間に技が出ないとテンポが死ぬ
   - しかしAI判定には時間がかかる
   
3. **日本語特化 vs 多言語対応？**
   - 「誉め殺し」は日本語文化に根ざした概念
   - しかし「Kill with kindness」は英語にもある
   
4. **対象は「相手」を褒めるのか「何でもいい」のか？**
   - 相手を褒める → 対人コミュニケーションの深さ
   - テーマお題を褒める → ボキャブラリー勝負

5. **課金ポイントはどこか？**
   - 基本無料 + キャラ/エフェクト課金？
   - 買い切り ¥100〜500？
   - 対戦チケット制？
