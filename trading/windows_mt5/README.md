# Windows native MT5 direct-validation workflow

## Goal

This directory is the answer to the failure mode you described:

- human exports a report
- code gets changed blindly
- human exports another report
- code becomes larger and harder to compile
- strategy still does not become robustly profitable

The intended replacement is:

- compile directly on Windows MT5
- run backtests directly on Windows MT5
- export audit CSV automatically
- analyze results with the Python tools in this repository
- make one controlled category of change at a time

## Recommended environment

Use one of these:

- Windows VPS
- Windows desktop

Do **not** use the Linux + Wine route as the primary production validation path.
Keep it only as a secondary path.

## Directory contents

- `compile_ea.ps1`
- `run_backtest.ps1`
- `run_opt_batch.ps1`
- `ascendant_tester.ini.tpl`
- `presets/`

## Included starting presets

These are **starting presets**, not production-approved final settings.

- `presets/SOVEREIGN_Ascendant_XAUUSD_M15_Hybrid.set`
  - lower base risk
  - wider spread tolerance
  - hybrid mode emphasis for gold
- `presets/SOVEREIGN_Ascendant_EURUSD_M15_Trend.set`
  - tighter spread tolerance
  - Bandwagon-forward trend bias
  - designed as the cleanest first validation target
- `presets/SOVEREIGN_Ascendant_USDJPY_M15_TrendStealth.set`
  - balanced trend/stealth profile
  - slightly more conservative than EURUSD on risk and session width

## Recommended validation order

If you want the fastest clean direct-validation path on Windows:

1. `EURUSD_M15_Trend`
2. `USDJPY_M15_TrendStealth`
3. `XAUUSD_M15_Hybrid`

Why this order:

- EURUSD usually gives the cleanest spread and execution conditions
- USDJPY is still relatively clean, but behaves differently enough to test robustness
- XAUUSD is the highest upside candidate, but also the noisiest execution environment

## Expected MT5 installation assumptions

Default assumptions:

- Terminal: `C:\MT5Portable\terminal64.exe`
- MetaEditor: `C:\MT5Portable\MetaEditor64.exe`
- Experts: `C:\MT5Portable\MQL5\Experts\`
- Tester presets: `C:\MT5Portable\MQL5\Profiles\Tester\`
- Reports: `C:\MT5Portable\reports\`

If your installation differs, pass explicit paths to the scripts.

## Recommended first-time setup on Windows

1. Install MT5 normally.
2. Log in to your broker account once.
3. Open the symbols you plan to test:
   - EURUSD
   - USDJPY
   - XAUUSD
4. Make sure history loads.
5. Copy `SOVEREIGN_Ascendant_v1.mq5` into `MQL5\Experts`.
6. Compile it with `compile_ea.ps1`.

## Compile example

```powershell
pwsh -File .\trading\windows_mt5\compile_ea.ps1 `
  -TerminalRoot "C:\MT5Portable" `
  -SourceFile "C:\path\to\repo\trading\mt5\SOVEREIGN_Ascendant_v1.mq5"
```

## Single backtest example

```powershell
pwsh -File .\trading\windows_mt5\run_backtest.ps1 `
  -TerminalRoot "C:\MT5Portable" `
  -ExpertName "SOVEREIGN_Ascendant_v1" `
  -SetFile "C:\path\to\repo\trading\windows_mt5\presets\SOVEREIGN_Ascendant_EURUSD_M15_Trend.set" `
  -Symbol "XAUUSD" `
  -Period "M15" `
  -FromDate "2025.01.01" `
  -ToDate "2025.03.31" `
  -ReportName "ascendant-xauusd-001"
```

For the included presets, pair the symbol properly:

- `SOVEREIGN_Ascendant_EURUSD_M15_Trend.set` -> `EURUSD`
- `SOVEREIGN_Ascendant_USDJPY_M15_TrendStealth.set` -> `USDJPY`
- `SOVEREIGN_Ascendant_XAUUSD_M15_Hybrid.set` -> `XAUUSD`

## Batch run example

This uses the `.set` files generated by the Python optimizer or any manually prepared `.set` files.

```powershell
pwsh -File .\trading\windows_mt5\run_opt_batch.ps1 `
  -TerminalRoot "C:\MT5Portable" `
  -ExpertName "SOVEREIGN_Ascendant_v1" `
  -SetFilesGlob "C:\path\to\repo\trading\windows_mt5\presets\*.set" `
  -Symbol "XAUUSD" `
  -Period "M15" `
  -FromDate "2025.01.01" `
  -ToDate "2025.03.31"
```

This PowerShell batch runner executes the `.set` files directly on Windows MT5 one by one.  
It is intentionally simple and stable:

- it does not depend on Wine
- it does not require the Linux runner
- it creates one report per `.set`
- it is suitable for the first direct-validation loop

Use one symbol-consistent batch at a time.
Do **not** mix EURUSD, USDJPY, and XAUUSD presets in one batch run unless you also vary `-Symbol` accordingly.

## How this reduces the painful loop

The old loop was:

- change code
- ask human to compile
- ask human to run
- ask human to export
- interpret manually

The new loop should be:

1. AI updates code in repo
2. Windows MT5 compiles directly
3. Windows MT5 backtests directly
4. audit CSV is generated automatically
5. Python analyzer ranks the result
6. only one controlled change is made next

## Mandatory process rules

To avoid the old failure mode, follow these rules:

1. Never make multiple unrelated strategy changes in one step.
2. Never accept a change without compile success.
3. Never optimize only for final profit.
4. Always inspect:
   - average R
   - PF
   - drawdown
   - trade count
   - mode-level expectancy
5. Disable weak modes instead of averaging them into the strategy.
6. Run OOS validation before promoting settings.

## Best workflow with the existing repository

Use this repository as:

- source of truth for `.mq5` strategy code
- source of truth for audit schema
- source of truth for Python scoring

Use the Windows MT5 machine as:

- direct compiler
- direct backtester
- direct report and audit producer

## What to send back after a Windows run

If you still want help improving the strategy after the direct-run environment is set up, the best artifacts to feed back are:

- generated audit CSV
- report `.htm`
- summary from `trading/sov_opt`

That is very different from the old manual loop because the execution path itself is now standardized and reproducible.
